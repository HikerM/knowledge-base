# AI Permission Policy

本文定义 AI 助手权限策略。当前阶段只做策略设计，不实现权限引擎，不接真实 AI，不暴露 mutation UI。

## 1. Default Decisions

默认允许：

- 搜索正式知识。
- 打开用户明确选择的文档。
- 总结当前打开文档。
- 解释当前打开文档。
- 生成整理建议、分类建议、checklist 草稿和 plan-only 建议。

必须确认：

- 保存长期记忆。
- 云端 AI 发送资料。
- 任何 safe execute 操作。
- 使用多个文档、非当前文档、未确认资料或敏感上下文进行 AI 处理。
- 生成会影响未来写操作的计划。

禁止：

- 自动删除。
- 自动归档。
- 自动恢复覆盖。
- 自动 promote。
- 自动修改 Markdown。
- 自动修改 SQLite。
- 自动清空资料。
- 自动发送敏感资料到云端。
- 自动保存长期记忆。
- 绕过 `knowledge_app.services` 直接读取文件、SQLite 或 runtime storage。
- 拼接 CLI 命令字符串执行能力。

## 2. Decision Matrix

| Capability level | Default decision | Confirmation | Execution |
| --- | --- | --- | --- |
| L0 read_only | allow | no | service read only |
| L1 ai_suggestion | allow local; confirm cloud/context expansion | context-dependent | response only |
| L2 plan_only | allow plan generation if scope is explicit | yes for high-risk plan | no execute |
| L3 safe_execute | confirm required | yes | TaskQueue after approval |
| L4 forbidden/destructive | deny | n/a | no execute |

## 3. Cloud AI Policy

Cloud provider use must be opt-in per operation unless the user has explicitly enabled a scoped policy later.

Before sending context to cloud:

- Show provider kind.
- Show context preview.
- Show document titles, layer, status, source type and citation ids.
- Show whether full body, snippets or metadata will be sent.
- Highlight raw, distilled, archived, sensitive, rejected or quarantine exclusions.
- Require explicit confirmation.

Cloud context preview must not be skipped because the user typed a natural-language request.

## 4. Sensitive Data Policy

Sensitive data includes but is not limited to:

- Secrets, credentials, tokens and private keys.
- Customer private data.
- Private business records.
- Unreviewed raw captures containing third-party material.
- Quarantine content.

Policy:

- Sensitive content is excluded by default.
- Secret-scan warnings should block cloud context until the user resolves or explicitly scopes the context.
- AI should summarize metadata when possible instead of sending full body.
- The preview must make excluded content visible as exclusion metadata, not silently drop it.

## 5. Memory Permission

Long-term memory requires explicit user confirmation.

Allowed without confirmation:

- Use short-term session state.
- Generate a `MemoryCandidateCard`.

Not allowed without confirmation:

- Save preference.
- Save workflow.
- Save personal rule.
- Save long-term goal.
- Share memory with cloud provider.

User controls required in future:

- View memory.
- Delete memory.
- Disable memory.
- Reject a memory candidate.
- Scope memory to workspace or device.

## 6. Safe Execute Permission

Safe execute is only for low-risk config-only operations that the backend already supports.

Required gates:

1. Registered L3 capability.
2. Plan generated by service.
3. Local snapshot / backup created.
4. User approval bound to exact plan hash.
5. TaskQueue task created.
6. Task executes through safe service.
7. Result and audit record returned.

If any gate fails, execution is denied.

Current L3 candidates:

- `update_category_display_name`
- `update_category_description`

Both must only affect `config/categories.yaml` display fields through existing safe mutation service. They must not rename paths, modify category id, modify Markdown or modify SQLite schema.

## 7. Destructive Policy

AI must never automatically execute:

- archive documents.
- delete documents.
- restore backup over current workspace.
- promote knowledge.
- template apply.
- merge categories.
- workspace delete/archive.
- cleanup tasks that delete runtime records.

Current response options:

- Explain that execution is forbidden.
- Offer read-only search to inspect affected documents.
- Offer future plan-only design if a plan service exists.
- Suggest backup/snapshot as a manual prerequisite.

## 8. Citation Policy

AI answers based on knowledge base content must include citations.

Citation minimum:

- document id or source path returned by service.
- title.
- layer.
- status.
- source type.
- confidence.
- chunk id or result id when available.

If content is unconfirmed, the answer must label it:

```text
未经审核，不能作为正式项目规则
```

No citation means the answer must be framed as a general assistant statement, not as a knowledge-base-backed fact.

## 9. Policy Output Shape

```json
{
  "decision": "allow|confirm|deny",
  "reason": "string",
  "required_cards": [
    "privacy_notice_card",
    "confirmation_card"
  ],
  "allowed_context": [],
  "blocked_context": [],
  "requires_task_queue": false,
  "requires_snapshot": false,
  "requires_approval": false
}
```
